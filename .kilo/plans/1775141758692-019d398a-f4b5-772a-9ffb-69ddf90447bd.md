# Реализация Backend для формы "Добавить еду"

## overview

Реализовать полный цикл обработки данных формы "Добавить еду" от Frontend до записи в БД согласно 8 этапам.

---

## Этапы реализации

### Этап 1: Frontend - Валидация и отправка (JS)

**Задачи:**
1. Добавить валидацию: проверить что все поля "Название" и "Вес" заполнены
2. Если пустое - показать `tg.showAlert()` с предупреждением
3. Упаковать данные в объект (название приема, массив продуктов)
4. Отправить POST запрос на `/api/save-meal`

**Файлы:**
- `public/assets/js/app.js` - функция `saveMeal()`

---

### Этап 2: Backend - анализ и сортировка (PHP)

**Задачи:**
1. Принять POST данные в контроллере
2. Разделить продукты на две группы:
   - `manualProducts` - где заполнены КБЖУ (ручной ввод)
   - `autoProducts` - где КБЖУ пустые (нужно получить через AI)

**Файлы:**
- `src/Controllers/AnalyzeController.php` - новый метод `saveMeal()`

---

### Этап 3: Получение КБЖУ через AI (PHP)

**Задачи:**
1. Для каждого продукта из `autoProducts` отправить запрос в OpenRouter API
2. Получить КБЖУ на 100г
3. Промпт для AI: "Верни JSON: {\"kcal\": число, \"protein\": число, \"fat\": число, \"carbs\": число} для 100г продукта [НАЗВАНИЕ]"

**Файлы:**
- `src/Services/OpenRouterService.php` - добавить метод `getProductNutrients()`

---

### Этап 4: Применение коэффициентов обработки (PHP)

**Задачи:**
1. Создать справочник коэффициентов:
   - Жарка → 1.4
   - Запекание → 1.3
   - Варка → 1.2 (или /3 для круп - упрощенно 1.2)
   - Тушение → 1.1
   - Не указано → 1.0
2. Применить коэффициент к каждому продукту

**Новый файл:**
- `src/Services/NutritionCalculatorService.php` - метод `applyProcessingCoefficient()`

---

### Этап 5: Расчет на порцию (PHP)

**Задачи:**
1. Для каждого продукта: КБЖУ_на_100г × вес_продукта / 100
2. Суммировать все КБЖУ по продуктам
3. Вернуть общие итоги

**Новый файл:**
- `src/Services/NutritionCalculatorService.php` - метод `calculateForMeal()`

---

### Этап 6: Запись в БД (PHP)

**Задачи:**
1. Открыть транзакцию
2. Создать общую запись в таблице `meals` (food_description = название приема, calories = итог, created_at = NOW())
3. Получить ID созданной записи
4. Создать детальные записи (или обновить если одна запись - упрощаем)
5. Зафиксировать транзакцию

**Файлы:**
- `src/Models/Meal.php` - метод `saveMealWithProducts()`
- `public/index.php` - добавить роут `/api/save-meal`

---

### Этап 7: Пересчет прогресса за день (PHP)

**Задачи:**
1. После сохранения сделать запрос: SUM(calories) WHERE user_id = ? AND DATE(created_at) = TODAY()
2. Вернуть итоговые калории за день

**Файлы:**
- `src/Models/User.php` - метод `getTodayCalories()`

---

### Этап 8: Ответ и обновление UI (JS + PHP)

**Задачи:**
1. Сервер возвращает: `{ status: 'success', today_calories: число, meal_id: число }`
2. Frontend закрывает модалку
3. Обновляет прогресс-бар (сегодняшние калории)
4. Добавляет запись в историю

**Файлы:**
- `public/assets/js/app.js` - обновить обработчик сохранения

---

## Структура данных

### Frontend → Backend
```json
{
  "meal_name": "Завтрак",
  "products": [
    {
      "name": "куриная грудка",
      "weight": 150,
      "processing": "fry",
      "kbju": { "kcal": 165, "protein": 31, "fat": 3.6, "carbs": 0 }
    }
  ]
}
```

### Backend → Frontend
```json
{
  "status": "success",
  "today_calories": 1850,
  "meal": {
    "description": "Завтрак",
    "calories": 450,
    "proteins": 45,
    "fats": 12,
    "carbs": 20
  }
}
```

## Порядок реализации

1. **JS** - валидация и отправка (Этап 1 + Этап 8)
2. **PHP контроллер** - прием данных (Этап 2)
3. **OpenRouterService** - получение КБЖУ (Этап 3)
4. **NutritionCalculatorService** - коэффициенты и расчет (Этап 4-5)
5. **Meal model** - запись в БД (Этап 6)
6. **Интеграция** - connected все вместе (Этап 7-8)
