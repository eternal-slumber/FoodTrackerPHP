# Plan: Attribute-Based Routing System

## Цель
Добавить атрибуты PHP 8 для маршрутизации, чтобы убрать ручную регистрацию маршрутов в `public/index.php` и объявить маршруты прямо в контроллерах.

## Принципы архитектуры (новичок-дружелюбно)
- **Явность**: маршруты видны прямо над методами контроллеров
- **Конвенция вместо конфигурации**: автодискавер через рефлексию, не нужно обновлять отдельный файл
- **Минимализм**: только необходимые классы (атрибут + лоадер), без сложных паттернов
- **Обратная совместимость**: Router остается почти неизменным

## Шаги реализации

### 1. Создать атрибут Route
**Файл**: `src/Attributes/RouteAttribute.php`

```php
namespace App\Attributes;

#[Attribute(Attribute::TARGET_METHOD)]
class RouteAttribute
{
    public function __construct(
        public readonly string $path,
        public readonly string $method = 'GET',
        public readonly ?string $name = null
    ) {}
}
```

### 2. Создать RouteLoader
**Файл**: `src/Services/RouteLoader.php` (или `src/Core/RouteLoader.php`)

Отвечает за:
- Сканирование указанных контроллеров (массив классов)
- Через рефлексию искать методы с `#[Route]`
- Регистрировать найденные маршруты в Router

```php
namespace App\Services;

use App\Attributes\RouteAttribute;
use App\Core\Router;
use ReflectionClass;
use ReflectionMethod;

class RouteLoader
{
    public function __construct(
        private Router $router,
        private array $controllerClasses
    ) {}

    public function load(): void
    {
        foreach ($this->controllerClasses as $controllerClass) {
            $reflection = new ReflectionClass($controllerClass);
            
            foreach ($reflection->getMethods(ReflectionMethod::IS_PUBLIC) as $method) {
                $attributes = $method->getAttributes(RouteAttribute::class);
                
                foreach ($attributes as $attribute) {
                    /** @var RouteAttribute $route */
                    $route = $attribute->newInstance();
                    $this->router->add($route->method, $route->path, $controllerClass, $method->getName());
                }
            }
        }
    }
}
```

### 3. Добавить атрибуты в контроллеры

**HomeController** (`src/Controllers/HomeController.php`):
```php
#[Route('/', 'GET')]
public function index(): void { ... }
```

**UserController** (`src/Controllers/UserController.php`):
```php

#[Route('/api/delete-profile', 'POST')]

```

**AnalyzeController** (`src/Controllers/AnalyzeController.php`):
```php

public function history(): string { ... }

```

### 4. Обновить public/index.php

Заменить ручную регистрацию:
```php
// Вместо 8 строк $router->add()...
$router->add('GET', '/', 'App\Controllers\HomeController', 'index');
// и т.д.
```

На:
```php
$routeLoader = new \App\Services\RouteLoader($router, [
    \App\Controllers\HomeController::class,
    \App\Controllers\UserController::class,
    \App\Controllers\AnalyzeController::class,
]);
$routeLoader->load();
```

Оставить все остальное без изменений.

## Почему так, а не иначе

**Почему RouteLoader как отдельный сервис, а не в Router?**
- SRP: Router только хранит и диспатчит, RouteLoader собирает
- Легко тестировать и заменять
- Можно добавить группировку или namespace-based сканирование позже

**Почему не автодискавер всех контроллеров в папке?**
- Для новичка явный массив классов понятнее (что включено, что нет)
- Позже можно расширить через конфиг

**Почему не Symfony/Routing компонент?**
- У learns цель — показать принципы АТРИБУТОВ, а не внедрять фреймворк
- Минимум зависимостей

## Риски и митигация

| Риск | Митигация |
|------|-----------|
| Рефлексия замедлит запуск | Кэширование routes в будущем, сейчас negligible |
| Ошибки в именах атрибутов | PHPStan/Tests (если есть) |
| Контроллеры с разными зависимостями | Container уже умеет разрешать через рефлексию |

## Расширения (будущее)

- `#[Route('/api/user/{id}', 'GET')]` + param extraction
- `#[Middleware(AuthMiddleware::class)]` для аутентификации
- Groups/prefixes `#[Group('/api')]`
- Кэширование сканирования в PHP файл
